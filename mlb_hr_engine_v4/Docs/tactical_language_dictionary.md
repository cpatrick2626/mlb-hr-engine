# Tactical Language Dictionary
## MLB HR Engine — Approved Terminology & Tone Doctrine

**Version:** 1.0  
**Date:** 2026-05-20  
**Phase:** Step 4/12 — Escalation Infrastructure & Pitcher Suppression Stabilization  
**Status:** Specification only. No runtime code modified.  
**Scope:** All user-facing text: card labels, caution explanations, tier words, tooltips, empty states, modal headers, notification text, error messages.

---

## Overview

Language is a system component. Inconsistent, informal, or hype-driven language breaks operator trust in the same way a misaligned badge color does. The MLB HR Engine is an intelligence tool — its words must be as precise as its probability model.

**Design principle:** Every word that appears in this platform should be consistent with how a professional quantitative analyst would describe the same signal. Not a journalist. Not a gambling influencer. Not a hype account.

---

## A. TONE PHILOSOPHY

### What This Platform Sounds Like

**Elite.** The language implies that the operator is a professional making decisions with machine-assisted intelligence — not someone being sold a pick.

**Measured.** Intensity is reserved for genuinely high-signal moments. When every phrase is urgent, nothing is. Urgency must be earned.

**Credible.** Claims are scoped. "HR probability is elevated" is credible. "This player is guaranteed to hit a home run" is not. The platform never overclaims.

**Machine-assisted.** The language reflects a system — signals, probabilities, modifiers, confidence. Not opinions, locks, or guarantees.

**Operational.** Phrases describe actions and states, not emotions. "Deploy," "confirm," "suppress," "escalate" — not "crush," "smash," "kill."

### What This Platform Does Not Sound Like

Not a gambling Discord at 1pm with the slate dropping.  
Not a Twitter account promising a 10-0 day.  
Not a sportsbook promotion for a same-game parlay.  
Not a YouTube thumbnail with fire emojis and "BEST PLAYS OF THE DAY."

## B. APPROVED TERMINOLOGY

These terms are standard. Use them precisely and consistently.

### Deployment & Action Terms

| Approved Term | Meaning | Example Use |
|---------------|---------|-------------|
| **deploy** | Execute a pick / place a bet | "Deploy this pick." "Deployment confidence: HIGH." |
| **deployment** | The act or state of executing a pick | "Deployment panel." "Ready for deployment." |
| **confirm** | Verify a signal before acting | "Confirm matchup before deployment." |
| **confirmation** | Signal verification state | "Awaiting confirmation." "Confirmation required." |
| **position** | Bet size or allocation | "Recommended position size." "Reduce position." |
| **position size** | Dollar or unit allocation for a pick | "Reduced position size indicated." |
| **slate** | Set of games/picks for a given date | "Today's slate." "Full Slate view." |

### Signal & Intelligence Terms

| Approved Term | Meaning | Example Use |
|---------------|---------|-------------|
| **escalation** | Movement to higher confidence tier | "Escalation tier: FIRE." "Escalation active." |
| **suppression** | Signal reducing deployment confidence | "Pitcher suppression: HIGH." "Suppression active." |
| **signal** | A data-derived intelligence indicator | "Suppression signal present." "No active signals." |
| **exploit** | A structural advantage in a matchup | "Pitch mix exploit identified." "Exploit signal: FAVORABLE." |
| **vulnerability** | A weakness in pitcher or batter profile | "Pitcher vulnerability to FB contact." |
| **threat** | A batter's deployment readiness level | "Threat tier: FIRE." "HR threat score." |
| **edge** | Positive expected value above market | "Edge: +4.2%." "Edge detection active." |
| **pressure** | Market-implied tension against model signal | "Market pressure on this line." |
| **tactical edge** | A specific identified advantage | "Tactical edge: platoon split + favorable park." |

### Risk & Caution Terms

| Approved Term | Meaning | Example Use |

## C. REJECTED TERMINOLOGY

These terms are permanently rejected. They associate the platform with gambling hype culture, amateur pick services, or consumer entertainment products. None belong in an intelligence tool.

### Locks & Guarantees

| Rejected | Why Rejected | Use Instead |
|----------|-------------|-------------|
| lock | Implies certainty. No pick is certain. | "High-confidence signal" |
| guaranteed | Same as lock. Dangerous overclaim. | "Calibrated probability: 89%" |
| can't lose | Irresponsible and inaccurate. | Never. |
| sure thing | Same as above. | Never. |
| mortal lock | Legacy gambling hype. | Never. |
| best bet | Implies preference without quantification. | "Top-tier pick by composite score" |

### Money & Gambling Slang

| Rejected | Why Rejected | Use Instead |
|----------|-------------|-------------|
| free money | Describes edge as riskless — it is not. | "Positive expected value detected" |
| smash spot | Influencer slang. Unprofessional. | "FIRE-tier deployment pick" |
| nuke | Same as smash spot. | "High-confidence exploit" |
| hammering | Gambling volume hype. | "Confirmed at full position size" |
| juice | Sometimes used for vig — confusing term. | "Market vig" or "no-vig probability" |
| fade | "Fading" a pick is vague and directional. | "Suppression active — caution warranted" |

### Action & Hype Language

| Rejected | Why Rejected | Use Instead |
|----------|-------------|-------------|
| fire away | Casino hype. | "Deploy." |
| pull the trigger | Same hype register. | "Confirm and deploy." |
| get on this | Urgency manipulation. | "Deployment window: tonight." |
| don't miss | Scarcity manipulation. | State the signal without urgency theater. |
| run it back | Sports-betting influencer phrase. | "Carry position to next eligible game." |
| capper | Gambling identity term. | "Operator." |

### Vague Superlatives

| Rejected | Why Rejected | Use Instead |
|----------|-------------|-------------|
| huge | Vague. Quantify instead. | "Barrel rate in 98th percentile" |
| monster | Same. | "Elite contact profile" |
| elite (as hype) | "Elite" is acceptable when supported by a threshold. "Elite" alone is hype. | "Elite barrel rate (≥12%) — top decile of MLB starters" |
| insane | Informal. | State the metric. |
| unreal | Same. | State the metric. |
| incredible | Same. | State the metric. |

## D. CONTEXTUAL LANGUAGE STANDARDS

### Card Labels

**Approved formats:**
```
FIRE TIER — PRIME DEPLOYMENT
HIGH SUPPRESSION ACTIVE
CAUTION: Pitcher Suppression — HIGH
MODERATE SUPPRESSION: Weak Contact Profile
No Suppression Signal
```

**Rejected formats:**
```
❌ THIS GUY IS ON FIRE
❌ SMASH THIS TONIGHT
❌ LOCK OF THE DAY
❌ AVOID THIS NIGHTMARE MATCHUP
❌ GET ON THIS NOW
```

### Caution Explanation Layer

**Approved format:**  
One to three declarative sentences. Subject + predicate. No exclamation. No alarmist phrasing.

```
Approved:
"Elite weak-contact profile. Pitcher's barrel-allowed rate ranks in the bottom 12th percentile 
of MLB starters. HR deployment confidence materially reduced for this matchup."

Rejected:
"WATCH OUT! This pitcher is ABSOLUTELY dominant and this matchup is DANGEROUS! 
Serious red flags here!"
```

### Tier Labels

**Approved:**  
FIRE, STRONG, WATCH, COLD, VOID (exact, always uppercase, never abbreviated)

**Rejected:**  
🔥HOT, ELITE LOCK, PRIME, DANGER, AVOID, SKIP, ⚠️WARNING, CRUSHER, SMASH

### Confidence Values

**Approved:**  
"Confidence: 87%" — plain numeric display  
"Low Sample — n < 30" — honest uncertainty label  
"Confidence: Elevated" — when numeric not available

**Rejected:**  
"Sky-high confidence!" — hype  
"Rock solid" — informal  
"Questionable" — ambiguous  

### Empty States

**Approved:**  
"No qualifying picks for today's slate."  
"Data unavailable — lineup not yet confirmed."  
"Filters returned no results for current parameters."

**Rejected:**  
"Nothing here yet... 👀"  
"Come back later!"  
"No locks available."

### Error Messages

**Approved:**  
"Data fetch failed. Retry or check connection."  
"Model output unavailable — dependency error in pipeline."

**Rejected:**  
"Uh oh! Something went wrong 😬"  
"Oops! We couldn't load your picks."

## E. ESCALATION TIER WORDING STANDARDS

When tier words appear in prose (not just badge labels), they follow these exact formats:

| Context | Format | Example |
|---------|--------|---------|
| Badge label | ALL CAPS | "FIRE" |
| Prose reference | Title case with qualifier | "FIRE-tier pick" |
| Explanation prose | Title case tier word only | "This is a FIRE pick based on..." |
| Short label | ALL CAPS, no qualifier | "FIRE — deploy" |
| Data table cell | ALL CAPS | "FIRE" |

Do not write:
- "fire pick" (lowercase — loses visual distinctiveness)
- "Fire Pick" (wrong capitalization pattern)
- "FIRE PICK" in body prose (caps in prose is shouting — use in badge/label context only)

## F. SUPPRESSION LANGUAGE STANDARDS

Suppression language must be precise without being alarming.

### Level-Appropriate Wording

| Suppression Tier | Card Header | Prose Explanation Tone |
|-----------------|-------------|----------------------|
| NONE | "No Suppression" | Neutral, no caution |
| LOW | "Minor Suppression Signal" | Note, no alarm |
| MODERATE | "Moderate Suppression Active" | Measured caution |
| HIGH | "High Suppression Active" | Direct caution |
| LOCKDOWN | "LOCKDOWN Suppressor" | Firm, clear, not alarming |

LOCKDOWN is the highest urgency phrase in the suppression vocabulary. It is direct. It is not alarming. It does not use exclamation marks, emojis, or flashing text.
